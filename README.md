# Velody — Платформа для поиска и аренды музыкантов

## Описание проекта

**Velody** — веб-платформа для поиска музыкантов различных направлений и публикации собственных услуг.

Проект решает проблему быстрого поиска исполнителей для музыкальных коллективов, организаторов мероприятий и отдельных музыкантов.

Пользователь может найти подходящего артиста по жанру, специализации, стоимости услуг и рейтингу, ознакомиться с профилем, отзывами и связаться с исполнителем.

Музыканты могут создать собственную карточку артиста, добавить информацию о себе, фотографии, направления деятельности и получать предложения от потенциальных клиентов.

---

# Основной функционал

## Для пользователей

- Регистрация и авторизация;
- Создание пользовательского профиля;
- Просмотр каталога музыкантов;
- Поиск артистов по:
  - жанрам;
  - специализации;
  - стоимости услуг;
  - опыту;
  - рейтингу;
- Просмотр карточки артиста;
- Добавление артистов в избранное;
- Отправка сообщений исполнителям;
- Просмотр отзывов и рейтинга.

---

## Для артистов

Авторизованный пользователь может создать профиль исполнителя.

В профиле доступны:

- сценическое имя;
- описание;
- город;
- опыт работы;
- стоимость услуг;
- музыкальные жанры;
- специализация;
- фотографии;
- рейтинг;
- отзывы.

Дополнительно реализовано:

- загрузка фотографий профиля;
- редактирование информации;
- публикация профиля;
- Premium-подписка.

---

# Premium-подписка

Для артистов реализована платная подписка через **ЮKassa**.

После успешной оплаты:

- в профиле сохраняется дата окончания подписки;
- пользователь получает статус активного Premium;
- подписку можно продлевать повторной оплатой.

# Внешние сервисы

## Timeweb S3 Storage

Используется для хранения фотографий артистов.

После загрузки изображения:

1. Файл отправляется в S3-хранилище;
2. Полученный URL сохраняется в базе данных;
3. Frontend использует URL для отображения фотографии.

---

## ЮKassa

Используется для обработки платежей Premium-подписки.

# Структура базы данных

## User

Пользователь платформы.

Основные поля:

- id
- email
- password
- role
- profile_id

---

## ArtistProfile

Профиль исполнителя.

Основные поля:

- id
- stage_name
- description
- city
- price_from
- experience_years
- rating
- reviews_count
- is_published
- subscription_expires_at

---

## ArtistPhoto

Фотографии артиста.

Поля:

- id
- artist_profile_id
- url
- order_index

---

## Genre

Музыкальные направления.

Примеры:

- рок;
- джаз;
- поп;
- классическая музыка.

---

## ArtistPosition

Специализация исполнителя.

Примеры:

- вокалист;
- гитарист;
- барабанщик;
- DJ.

---

## Review

Отзывы пользователей.

Поля:

- id
- rating
- comment
- artist_profile_id
- author_profile_id

---

## UserFavorites

Избранные артисты пользователей.

# Пользовательские сценарии

## Клиент

1. Открывает каталог артистов.
2. Использует фильтры поиска.
3. Открывает карточку исполнителя.
4. Изучает:
   - описание;
   - фотографии;
   - рейтинг;
   - отзывы.
5. Связывается с артистом.

---

## Артист

1. Создает аккаунт.
2. Создает профиль исполнителя.
3. Добавляет информацию о себе.
4. Загружает фотографии.
5. Публикует профиль.
6. Получает предложения от клиентов.
7. Подключает Premium для дополнительного продвижения.
